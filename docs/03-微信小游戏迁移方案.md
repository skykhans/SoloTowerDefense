# 微信小游戏迁移方案

## 迁移目标

将当前 Web 塔防原型迁移为基于 Cocos Creator 的微信小游戏工程，并保留现有核心数值与战斗规则。

## 迁移原则

- 先拆分逻辑层与表现层
- 保留数值规则，重建 UI 与渲染方式
- 先完成 P0 最小可玩闭环，再扩展内容

## 可复用模块

- 炮塔配置
- 敌人配置
- 波次配置
- 选敌与攻击逻辑
- 敌人移动与到终点扣血逻辑
- 胜负与资源结算逻辑

## 需要重写模块

- DOM 布局与按钮交互
- Canvas 直接绘制逻辑
- 浏览器键鼠事件
- 本地静态服务器启动方案

## 建议的 Cocos 目录结构

```text
assets/
  scenes/

```

## 迁移阶段

### 第一阶段：战斗骨架

- 主场景
- 路径点
- 敌人移动
- 两种塔攻击
- 子弹伤害
- 波次生成
- 胜负判定

### 第二阶段：内容扩展

- 补齐更多塔与敌人
- 增加 Boss 技能和战斗反馈
- 加入多地图与章节
- 强化音效和特效

### 第三阶段：产品闭环

- 主界面
- 关卡选择
- 结算页
- 新手引导
- 本地存档
- 星级与解锁

### 第四阶段：微信接入

- 导出微信小游戏
- 真机性能优化
- 分辨率适配
- 激励广告
- 分享与复活

## 迁移优先级建议

1. 保住逻辑闭环
2. 再做 UI 和资源化
3. 最后接平台能力
