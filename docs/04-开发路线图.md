# 开发路线与任务拆解

## Web 原型当前开发重点

为了在迁移到微信小游戏前继续验证玩法，Web 原型继续承担以下任务：

- 验证塔与敌人的战斗数值
- 验证波次节奏与局内资源曲线
- 验证 UI 信息密度是否合适
- 验证主动技能和结算反馈是否增强体验

## 当前 Web 原型后续优先级

### P0

- 失败结算信息完善
- 最佳成绩记录
- 局内技能
- 更清晰的新手提示

### P1

- 第三种塔
- 第四类特殊敌人
- 音效与命中特效
- 简单关卡或地图切换

### P2

- 局外成长占位
- 成就与任务占位
- 更多技能与卡片随机事件

## Cocos 版本按周计划

### 第 1 周

- 初始化项目
- 建立 `GameState`
- 实现路径与敌人移动
- 实现基础战斗

### 第 2 周

- 实现建塔、升级、出售
- 实现波次系统
- 实现局内 UI
- 做第一张地图

### 第 3 周

- 扩展塔和敌人种类
- 加结算和关卡入口
- 加存档

### 第 4 周

- 导出微信小游戏
- 真机适配
- 包体与性能优化
- 接入激励广告

## 里程碑定义

- M1：可玩原型
- M2：完整单局
- M3：具备产品闭环
- M4：微信可测试版本
