# 数据结构说明

## 数据结构分层

当前项目的数据结构分为四层：

1. 静态配置数据
2. 运行时状态数据
3. 实体数据
4. 本地存档数据
5. 跨场景会话数据

## 静态配置数据

### 章节配置 `ChapterDefinition`

位置：`assets/scripts/config/ChapterConfig.ts`

字段：

- `id`
  - 章节编号
- `name`
  - 章节名称
- `description`
  - 章节说明
- `levelIds`
  - 当前章节包含的关卡编号列表

### 炮塔配置 `TowerDefinition`

位置：`assets/scripts/config/TowerConfig.ts`

字段：

- `id`
  - 炮塔类型标识
- `name`
  - 炮塔名称
- `cost`
  - 建造消耗
- `range`
  - 攻击范围
- `fireInterval`
  - 攻击间隔
- `damage`
  - 单次伤害
- `projectileSpeed`
  - 子弹速度
- `splashRadius`
  - 溅射半径，可选
- `slowFactor`
  - 减速倍率，可选
- `slowDuration`
  - 减速持续时间，可选

### 敌人配置 `EnemyDefinition`

位置：`assets/scripts/config/EnemyConfig.ts`

字段：

- `id`
  - 敌人类型标识
- `name`
  - 敌人名称
- `hpFactor`
  - 血量倍率
- `speedFactor`
  - 速度倍率
- `rewardBonus`
  - 额外奖励
- `lifeDamage`
  - 漏怪后扣除生命值
- `radius`
  - 碰撞半径或表现半径
- `shieldFactor`
  - 护盾倍率，可选，用于生成额外护盾值
- `splitInto`
  - 死亡后分裂出的敌人类型，可选
- `splitCount`
  - 死亡后分裂出的敌人数，可选
- `healFactor`
  - 治疗倍率，可选，用于生成单次治疗量
- `healRadius`
  - 治疗范围，可选
- `healInterval`
  - 治疗间隔，可选
- `isBoss`
  - 是否 Boss

### 波次配置

### 关卡配置

推荐字段：

- `id`
  - 关卡编号
- `name`
  - 关卡名称
- `chapterId`

- `startingLives`

  - 初始金币

## 实体数据

### 敌人实体 `Enemy`

位置：`assets/scripts/entities/Enemy.ts`

字段：

- `enemyType`
- `hp`
- `maxHp`
- `baseSpeed`
- `reward`
- `lifeDamage`
- `shieldHp`
- `maxShieldHp`
- `splitInto`
- `splitCount`
- `hasSplit`
- `healAmount`
- `healRadius`
- `healInterval`
- `healTimer`
- `waypointIndex`
- `slowFactor`
- `slowTimer`

说明：

- `Enemy` 负责战斗数值、移动状态和减速状态
- 护盾怪会优先消耗 `shieldHp`，护盾打空后才会开始扣除生命值
- 分裂怪死亡后会按 `splitInto + splitCount` 生成后续小怪
- 治疗怪会按 `healInterval` 周期检查附近单位，并恢复生命值

## 跨场景会话数据

### 战斗结果会话 `BattleResultData`

位置：`assets/scripts/core/BattleResultSession.ts`

字段：

- `result`
  - 结算类型，胜利或失败
- `levelId`
  - 结算对应关卡编号
- `reachedWave`
  - 本局到达波次
- `kills`
  - 本局击杀
- `bestWave`
  - 本局结束时的最佳波次记录
- `stars`
  - 本局星级评价
- `awardedCoins`
  - 本局结算获得金币
- `firstClearReward`
  - 本局结算中的首次通关奖励金币
- `unlockedLevelId`
  - 本局结算后新解锁的关卡编号，可为空

### 首页提示会话 `HomeNoticeSession`

位置：`assets/scripts/core/HomeNoticeSession.ts`

说明：

- 用于把结算场景中的摘要提示回传到首页
- 当前为轻量字符串会话
- 首页读取一次后即消费并清空

## 首页派生数据

### 首页进度服务 `HomeProgressService`

位置：`assets/scripts/core/HomeProgressService.ts`

职责：

- 统计已通关关卡数量
- 统计历史总星级
- 计算推荐进入的关卡编号
- 统一首页开始按钮文案与关卡进度摘要文本

## 当前数据流

1. 启动游戏时读取 `PlayerSaveData`
2. 将 `bestWave` 等关键数据同步到 `GameState`
3. 首页通过 `HomeUI` 读写当前选择关卡
4. 开始波次时，`WaveSpawnSystem` 读取 `WaveDefinition`
5. 战斗时读取静态配置生成敌人、炮塔与波次
6. 战斗结束时先把结算写入 `BattleResultSession`
7. 结算场景读取会话数据并写回本地存档
