# 配置管理说明

## 当前是否有管理员配置后台

当前没有传统意义上的管理员后台，也没有独立后端服务。

现阶段项目采用的是：

- `Cocos Creator + TypeScript`
- 无后端开发
- 配置文件驱动的关卡与数值管理

也就是说，当前的“配置后台”是代码侧的配置层，而不是网页管理系统。

## 当前配置管理方式

配置文件位置：

```text
cocos-wechat/assets/scripts/config/
  ConfigService.ts
  ChapterConfig.ts
  TowerConfig.ts
  EnemyConfig.ts
  WaveConfig.ts
  LevelConfig.ts
```

当前可配置内容：

- 炮塔类型、费用、伤害、攻速、射程、特殊效果
- 章节结构、章节说明与关卡归属
- 敌人类型、血量倍率、速度倍率、奖励、漏怪扣血
- 波次敌人组成与生成规则
- 关卡初始生命、初始金币、建造点位置、起始波次

当前统一读取入口：

- `cocos-wechat/assets/scripts/config/ConfigService.ts`

作用：

- 为业务层提供统一配置访问方式
- 避免系统层直接散乱依赖多个配置文件
- 为未来后台、配置工具、JSON 化或远程配置接入预留统一入口

## 为什么当前先不用管理员后台

当前仍处于玩法和系统搭建阶段，优先级是：

1. 把微信小游戏主线版本跑通
2. 让核心数据结构稳定
3. 再决定是否需要独立配置后台

如果过早做后台，会出现两个问题：

- 核心数据结构频繁变化，后台容易反复重构
- 没有后端时，后台改完的数据也缺少统一持久化链路

## 当前推荐的配置流程

### 炮塔配置

修改：

- `cocos-wechat/assets/scripts/config/TowerConfig.ts`

可控制：

- 费用
- 射程
- 攻速
- 伤害
- 子弹速度
- 溅射
- 减速

### 敌人配置

修改：

- `cocos-wechat/assets/scripts/config/EnemyConfig.ts`

可控制：

- 血量倍率
- 速度倍率
- 奖励
- 漏怪伤害
- 护盾倍率
- Boss 标记

### 章节配置

修改：

- `cocos-wechat/assets/scripts/config/ChapterConfig.ts`

可控制：

- 章节名
- 章节说明
- 章节包含的关卡列表

### 波次配置

修改：

- `cocos-wechat/assets/scripts/config/WaveConfig.ts`

可控制：

- 每波敌人种类
- 数量
- 出兵间隔
- Boss 波逻辑

### 关卡配置

修改：

- `cocos-wechat/assets/scripts/config/LevelConfig.ts`

可控制：

- 关卡名
- 初始金币
- 初始生命
- 建造点位置
- 起始波次

## 后续如果要做管理员后台，建议分阶段

### 第一阶段：编辑器内配置化

- 继续完善 TS / JSON 配置
- 在 Cocos 中支持更方便的关卡编辑

### 第二阶段：无后端本地配置工具

- 做一个内部配置页面或独立工具
- 输出 JSON 配置文件
- 游戏构建时导入配置

### 第三阶段：真正后台

前提：

- 已接后端
- 已有账号体系
- 已有配置发布流程

届时可以做：

- 关卡编辑后台
- 数值表后台
- 活动配置后台
- 任务奖励后台
- 热更新配置发布

## 当前结论

当前没有管理员后台。

当前已有的是“可维护的配置层”，适合无后端阶段继续开发。

等 Cocos 版本主流程、场景、预制体和微信平台接入稳定后，再评估是否要做独立后台最合适。
