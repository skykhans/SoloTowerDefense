# 部署说明

## 部署范围

当前仓库包含两条线：

- `仓库根目录`
  - 主开发方向
  - 目标部署到微信小游戏
- `legacy-web/`
  - 旧版 Web 原型
  - 仅用于本地验证和历史参考

## 一、Cocos 微信小游戏版部署

## 1. 开发环境准备

需要安装：

- Cocos Creator 3.x
- 微信开发者工具
- Node.js

## 2. 打开项目

当前仓库中的 Cocos 项目目录为：

```text
SoloTowerDefense/
```

根目录提供了主线一键启动脚本：

- `start.mjs`
- `start.bat`
- `start.ps1`

脚本行为：

- 若检测到常见路径或环境变量指定的 Cocos Creator，则尝试直接打开仓库根目录
- 若未检测到，则打开项目目录和主场景接入文档

推荐在 macOS / Linux / 已安装 Node 的环境直接运行：

```bash
npm start
```

## 3. 推荐开发流程

1. 使用 Cocos Creator 打开当前仓库根目录
2. 按 [Cocos场景搭建清单](./18-Cocos场景搭建清单.md) 检查 `HomeScene / MainScene / ResultScene`
3. 创建并绑定敌人、炮塔、建造点、子弹、命中特效预制体
4. 完成 `Installer` Inspector 绑定
5. 运行并检查场景跳转、战斗主流程、结算主流程

## 4. 微信小游戏导出流程

在 Cocos Creator 中：

1. 选择构建发布
2. 平台选择“微信小游戏”
3. 配置包名、方向、资源策略
4. 执行构建
5. 用微信开发者工具打开构建产物目录

## 5. 当前无后端部署特征

当前版本是无后端方案，因此部署时：

- 不需要 API 服务
- 不需要数据库
- 不需要云函数
- 用户数据存本地

## 二、旧版 Web 原型部署

旧版原型目录：

```text
legacy-web/
```

进入目录后可运行：

```bash
cd legacy-web
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 三、部署建议

1. 主线开发与后续发布都围绕仓库根目录中的 Cocos 工程
2. `legacy-web/` 不再继续作为正式版本部署目标
3. 等 Cocos 版完整覆盖旧版原型后，再考虑移除 `legacy-web/`
