# 部署说明

## 部署范围

当前仓库包含两条线：

- `cocos-wechat/`
  - 主开发方向
  - 目标部署到微信小游戏
- `legacy-web/`
  - 旧版 Web 原型
  - 仅用于本地验证和历史参考

## 一、Cocos 微信小游戏版部署

## 1. 开发环境准备

需要安装：

- Cocos Creator 3.x
- 微信开发者工具
- Node.js

建议版本策略：

- Cocos Creator 使用稳定版 3.x
- 微信开发者工具使用最新稳定版
- Node.js 使用 LTS 版本

## 2. 打开项目

当前仓库中的 Cocos 代码目录为：

```text
cocos-wechat/
```

说明：

- 当前目录已具备脚本骨架
- 已包含 `assets/scenes / prefabs / textures / audio` 资源目录占位
- 但还不是完整的 Cocos 工程目录
- 需要在 Cocos Creator 中创建或初始化场景、预制体、资源和工程元文件

根目录提供了主线一键启动脚本：

- `start.bat`
- `start.ps1`

脚本行为：

- 若检测到常见路径或环境变量指定的 Cocos Creator，则尝试直接打开 `cocos-wechat/`
- 若未检测到，则打开项目目录和主场景接入文档

可选环境变量：

- `COCOS_CREATOR_EXE`
- `COCOS_CREATOR_PATH`

它们都可以直接指向本机的 `CocosCreator.exe`

如当前机器尚未安装编辑器，可先参考：

- [28-CocosCreator安装与首次打开说明.md](/C:/Test/SoloTowerDefense/docs/28-CocosCreator安装与首次打开说明.md)

## 3. 推荐开发流程

1. 使用 Cocos Creator 创建或打开项目目录 `cocos-wechat/`
2. 按 [Cocos场景搭建清单](/C:/Test/SoloTowerDefense/docs/18-Cocos场景搭建清单.md) 创建 `HomeScene / MainScene / ResultScene`
3. 创建并绑定：
   - 敌人预制体
   - 炮塔预制体
   - 建造点预制体
   - 子弹预制体
   - 命中特效预制体
4. 完成 `Installer` Inspector 绑定
5. 运行并检查场景跳转、战斗主流程、结算主流程

## 4. 微信小游戏导出流程

在 Cocos Creator 中：

1. 选择构建发布
2. 平台选择“微信小游戏”
3. 配置包名、方向、资源策略
4. 执行构建
5. 用微信开发者工具打开构建产物目录

推荐联动阅读：

- [微信小游戏构建配置说明](/C:/Test/SoloTowerDefense/docs/19-微信小游戏构建配置说明.md)
- [微信小游戏发布前检查清单](/C:/Test/SoloTowerDefense/docs/20-微信小游戏发布前检查清单.md)

## 5. 当前无后端部署特征

当前版本是无后端方案，因此部署时：

- 不需要 API 服务
- 不需要数据库
- 不需要云函数
- 用户数据存本地

当前本地存储方式：

- 微信小游戏环境：通过 `PlatformService` 走 `wx` 本地存储
- Web 环境：通过 `PlatformService` 回退到浏览器本地存储
- 兜底环境：通过 `PlatformService` 回退到内存存储

## 6. 当前不包含的发布项

当前还没有：

- 正式小游戏 `appId`
- 微信登录
- 云开发
- 排行榜
- 激励广告
- 线上配置后台

## 二、旧版 Web 原型部署

旧版原型目录：

```text
legacy-web/
```

用途：

- 本地演示
- 数值验证
- 历史参考

## 1. 本地启动方式

进入目录：

```bash
cd legacy-web
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

Windows 下也可以直接运行：

- `legacy-web/start.bat`
- `legacy-web/start.ps1`

## 2. 静态托管方式

如果只想临时演示旧版原型，可以部署到任意静态站点：

- Nginx 静态目录
- GitHub Pages
- Vercel
- Netlify

说明：

- 旧版原型没有后端依赖
- 只需托管静态文件即可

## 三、部署建议

当前推荐策略：

1. 主线开发与后续发布都围绕 `cocos-wechat/`
2. `legacy-web/` 不再继续作为正式版本部署目标
3. 等 Cocos 版完成场景、预制体和微信导出闭环后，再考虑移除旧版原型
