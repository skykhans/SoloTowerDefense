# 预制体接入说明

## 目标

本文档说明当前 Cocos 主线需要的预制体、推荐组件挂载方式，以及与现有脚本的绑定关系。

## 当前需要的预制体

```text
prefabs/
  Enemy.prefab
  Tower.prefab
  BuildSpot.prefab
  Bullet.prefab
  HitEffect.prefab
```

推荐联动阅读：

- [16-预制体视觉结构规范.md](./16-预制体视觉结构规范.md)
- [29-预制体结构自检说明.md](./29-预制体结构自检说明.md)

## 1. Enemy 预制体

建议名称：

- `Enemy.prefab`

建议结构：

```text
Enemy
  Sprite
  HpBar
```

推荐挂载脚本：

- `Enemy`
- `EnemyView`

说明：

- `GameManager.spawnEnemy()` 会实例化它
- 生成后会调用 `Enemy.setup(...)`
- `EnemyView` 推荐提供：
  - `normalVisual`
  - `fastVisual`
  - `heavyVisual`
  - `shieldVisual`
  - `splitVisual`
  - `healerVisual`
  - `bossVisual`
  - `hpBarFill`
  - `slowTintTarget`
- 当前会自动同步敌人血条长度、类型外观和减速态染色
- 护盾怪若暂时没有独立节点，会自动回退复用 `heavyVisual`
- 分裂怪若暂时没有独立节点，会自动回退复用 `fastVisual`
- 治疗怪若暂时没有独立节点，会自动回退复用 `normalVisual`

## 2. Tower 预制体

建议名称：

- `Tower.prefab`

建议结构：

```text
Tower
  BaseSprite
  CoreSprite
```

推荐挂载脚本：

- `Tower`
- `TowerView`

说明：

- `GameManager.createTower()` 会实例化它
- 生成后会调用 `Tower.setup(...)`
- `TowerView` 推荐提供：
  - `selectionRing`
  - `rangeIndicator`
  - `rapidVisual`
  - `cannonVisual`
  - `frostVisual`
- 当前选择炮塔后，范围圈和选中高亮会自动显示

## 3. BuildSpot 预制体

建议名称：

- `BuildSpot.prefab`

建议结构：

```text
BuildSpot
  Sprite
  Highlight
```

推荐挂载脚本：

- `BuildSpot`
- `BuildSpotClickHandler`
- `BuildSpotView`

说明：

- `BuildSpotManager.loadLevel()` 会实例化它
- 当前最少需要能显示“可建造位置”
- 点击后可通过 `BuildSpotClickHandler` 调用 `GameManager.createTower()`
- `BuildSpotManager` 会在生成时把 `GameManager` 和 `GameState` 注入到 `BuildSpotClickHandler`
- `BuildSpotView` 推荐提供：
  - `availableVisual`
  - `occupiedVisual`
  - `buildModeHintVisual`
- 当前进入建造模式后，空闲建造点会自动显示建造提示

## 4. Bullet 预制体

建议名称：

- `Bullet.prefab`

建议结构：

```text
Bullet
  Sprite
```

推荐挂载脚本：

- `Bullet`
- `BulletView`

说明：

- `TowerCombatSystem` 会实例化它并驱动飞行
- `BulletView` 推荐提供：
  - `rapidVisual`
  - `cannonVisual`
  - `frostVisual`
- 当前会根据来源炮塔自动切换子弹外观

## 5. HitEffect 预制体

建议名称：

- `HitEffect.prefab`

建议结构：

```text
HitEffect
  CoreVisual
```

推荐挂载脚本：

- `HitEffect`

说明：

- 当前用于命中时的轻量占位特效
- 推荐挂到 `EffectLayer`

## Inspector 关键绑定

### `GameManager`

需要绑定：

- `enemyLayer`
- `towerLayer`
- `enemyPrefab`
- `towerPrefab`

说明：

- `gameState`
- `pathManager`
- `buildSpotManager`
- `waveSpawnSystem`
- `enemyMovementSystem`
- `towerCombatSystem`

推荐统一由 `MainSceneInstaller` 注入，而不是在多个节点手工重复绑定

说明：

- `MainSceneInstaller` 还会校验 `Enemy.prefab / Tower.prefab / BuildSpot.prefab / Bullet.prefab` 是否挂载了基础脚本组件

### `WaveSpawnSystem`

当前无需额外绑定资源节点，但必须挂入场景并在 `GameManager` Inspector 中引用。

### `EnemyMovementSystem`

说明：

- `pathManager` 推荐由 `MainSceneInstaller` 注入

### `TowerCombatSystem`

需要绑定：

- `bulletLayer`
- `bulletPrefab`

可选增强绑定：

- `effectLayer`
- `hitEffectPrefab`

说明：

- 如果配置了 `effectLayer + hitEffectPrefab`，命中时会自动生成轻量特效

### `BuildSpotManager`

需要绑定：

- `buildSpotPrefab`
- `buildSpotLayer`

说明：

- `gameManager` 推荐由 `MainSceneInstaller` 注入

### `BattleUI`

需要绑定：

- `livesLabel`
- `goldLabel`
- `waveLabel`
- `killsLabel`
- `bestWaveLabel`
- `phaseLabel`
- `buildModeLabel`
- `startWaveButton`
- `startWaveButtonLabel`
- `pauseButton`
- `pauseButtonLabel`
- `tutorialLabel`（可选）

说明：

- `gameManager` 与 `gameState` 推荐由 `MainSceneInstaller` 注入
- 如场景中有教程提示区，推荐额外绑定 `tutorialLabel`

可给按钮绑定的方法：

- `onSelectRapid`
- `onSelectCannon`
- `onSelectFrost`
- `onClearBuildSelection`
- `onStartWave`
- `onTogglePause`
- `onRestartBattle`

说明：

- 选择建造模式时会自动清除当前选中炮塔，避免范围圈和建造模式冲突
- 开始波次按钮会根据当前阶段自动切换文案和可点击状态
- 暂停按钮会根据当前暂停状态自动切换文案

### `HomeUI`

需要绑定：

- `summaryLabel`
- `levelLabel`
- `levelDetailLabel`
- `statusLabel`
- `prevLevelButton`
- `nextLevelButton`
- `startLevelButton`
- `startLevelButtonLabel`
- `shareButton`
- `shareButtonLabel`

可给按钮绑定的方法：

- `onPrevLevel`
- `onNextLevel`
- `onSelectCurrentLevel`
- `onStartSelectedLevel`
- `onResetProgress`
- `onShareGame`

推荐接法：

- 将 `HomeUI` 挂到首页根节点
- 再由 `HomeSceneInstaller` 校验标签绑定

### `ResultUI`

需要绑定：

- `panelRoot`
- `resultLabel`
- `titleLabel`
- `actionHintLabel`
- `retryButton`
- `backHomeButton`
- `shareButton`
- `shareButtonLabel`

可给按钮绑定的方法：

- `onBackHome`
- `onRetryBattle`
- `onShareResult`

推荐接法：

- 将 `ResultUI` 挂到结算根节点
- 再由 `ResultSceneInstaller` 校验 `panelRoot` 与 `resultLabel`

### `TowerPanel`

需要绑定：

- `gameManager`
- `titleLabel`
- `infoLabel`
- `upgradeValueLabel`
- `sellValueLabel`
- `actionHintLabel`
- `upgradeButton`
- `sellButton`

可给按钮绑定的方法：

- `onUpgradeTower`
- `onSellTower`

说明：

- 未选中炮塔时，两个按钮会自动禁用
- 金币不足时，升级按钮会自动禁用

### `PlatformDebugPanel`

需要绑定：

- `panelRoot`
- `infoLabel`

说明：

- 推荐只在开发期场景中挂载
- 用于快速确认平台识别、分辨率和语言信息

## Installer 推荐绑定

- `HomeSceneInstaller`
  - 绑定 `homeUI`
- `MainSceneInstaller`
  - 绑定 `gameState`
  - 绑定 `gameManager`
  - 绑定 `pathManager`
  - 绑定 `buildSpotManager`
  - 绑定 `waveSpawnSystem`
  - 绑定 `enemyMovementSystem`
  - 绑定 `towerCombatSystem`
  - 绑定 `battleUI`
  - 绑定 `towerPanel`
- `ResultSceneInstaller`
  - 绑定 `resultUI`

## 当前推荐优先级

1. 先做 `BuildSpot.prefab`
2. 再做 `Enemy.prefab`
3. 再做 `Tower.prefab`
4. 最后补 `Bullet.prefab`
