# 场景流转与接线规范

## 目标

本文档说明当前 `cocos-wechat` 主线推荐采用的场景流转方式，以及各场景推荐使用的 Installer 接线脚本。

## 场景流转

```text
HomeScene -> MainScene -> ResultScene
     ^            |            |
     |            |            |
     +------------+------------+
```

- `HomeScene`
  - 负责首页展示、关卡选择、开始战斗
- `MainScene`
  - 负责单局战斗、刷怪、建塔、升级、出售、胜负判定
- `ResultScene`
  - 负责战斗结算、返回首页、再次挑战

## 当前推荐原则

- 每个场景只负责自己的 UI 和状态展示
- 跨场景数据只通过轻量会话对象传递
- Inspector 绑定由 Installer 脚本统一校验
- `GameManager` 只处理战斗域逻辑，不直接控制首页和结算场景 UI

## 场景跳转服务

统一使用：

- `SceneNames.ts`
- `SceneNavigator.ts`

不要在业务脚本中直接写死字符串场景名。

## 战斗结果传递

单局结束后：

1. `GameManager` 组装结算数据
2. 写入 `BattleResultSession`
3. 调用 `SceneNavigator.goTo(SCENE_NAMES.result)`
4. `ResultUI` 在新场景中读取并消费结算数据

这样可以避免：

- `MainScene` 长时间持有结算 UI 依赖
- 场景销毁后对象引用失效
- 结算逻辑和战斗主流程强耦合

## Installer 设计

推荐每个场景都在根节点挂一个 Installer 脚本：

- `HomeSceneInstaller`
- `MainSceneInstaller`
- `ResultSceneInstaller`

## Installer 职责

- 校验本场景必须存在的脚本和节点绑定
- 校验根节点命名和关键结构节点是否已创建
- 为 UI 脚本注入 `GameManager`、`GameState` 等依赖
- 为系统脚本建立必要引用
- 在缺少必要绑定时尽早抛错

## 当前绑定校验工具

统一使用：

- `SceneBindingValidator.ts`
- `SceneStructureValidator.ts`

适用场景：

- 根节点名称不正确
- 关键结构节点未创建
- 关键组件为空
- 关键节点未绑定
- UI 文本标签遗漏
- 预制体或 Layer 未配置

## MainScene 推荐接线顺序

1. 创建 `Managers / World / UI`
2. 挂 `GameState`
3. 挂 `GameManager`
4. 挂 `PathManager`
5. 挂 `BuildSpotManager`
6. 挂 `WaveSpawnSystem`
7. 挂 `EnemyMovementSystem`
8. 挂 `TowerCombatSystem`
9. 挂 `BattleUI`
10. 挂 `TowerPanel`
11. 如需开发调试，挂 `PlatformDebugPanel`
12. 挂 `MainSceneInstaller`
13. 在 `MainSceneInstaller` Inspector 中绑定上述对象
14. 确认 `TowerPanel` 的标题、详情、花费文案和按钮字段已全部绑定

## HomeScene 推荐接线顺序

1. 创建首页节点结构
2. 挂 `HomeUI`
3. 挂 `HomeSceneInstaller`
4. 在 Installer 中绑定 `HomeUI`
5. 确认 `prevLevelButton / nextLevelButton / startLevelButton / startLevelButtonLabel` 已绑定
6. 确认 `shareButton / shareButtonLabel` 已绑定

## ResultScene 推荐接线顺序

1. 创建结算节点结构
2. 挂 `ResultUI`
3. 挂 `ResultSceneInstaller`
4. 在 Installer 中绑定 `ResultUI`
5. 确认 `titleLabel / actionHintLabel / retryButton / backHomeButton` 已绑定
6. 确认 `shareButton / shareButtonLabel` 已绑定

## 当前收益

- 绑定入口更集中
- 缺字段更早暴露
- 文档和 Inspector 的对应关系更清晰
- 场景职责边界更稳定
