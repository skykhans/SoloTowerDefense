# 预制体视觉结构规范

## 目标

本文档定义当前主线推荐采用的预制体视觉层级，便于在 Cocos Creator 中快速搭建统一结构，并和现有 `View` 组件对齐。

## 1. Enemy.prefab

推荐结构：

```text
Enemy
  Body
    NormalVisual
    FastVisual
    HeavyVisual
    ShieldVisual
    SplitVisual
    HealerVisual
    BossVisual
  HpBarRoot
    HpBarBg
    HpBarFill
```

推荐挂载：

- `Enemy`
- `EnemyView`

说明：

- `NormalVisual / FastVisual / HeavyVisual / ShieldVisual / SplitVisual / HealerVisual / BossVisual`
  - 对应 `EnemyView` 的敌人视觉节点
- `HpBarFill`
  - 对应 `EnemyView.hpBarFill`
- `Body` 的 `Sprite`
  - 可直接绑定为 `slowTintTarget`

说明：

- `ShieldVisual` 暂时没有资源时，可先不创建，代码会回退显示 `HeavyVisual`
- `SplitVisual` 暂时没有资源时，可先不创建，代码会回退显示 `FastVisual`
- `HealerVisual` 暂时没有资源时，可先不创建，代码会回退显示 `NormalVisual`

## 2. Tower.prefab

推荐结构：

```text
Tower
  Base
  VisualRoot
    RapidVisual
    CannonVisual
    FrostVisual
  SelectionRing
  RangeIndicator
```

推荐挂载：

- `Tower`
- `TowerView`

说明：

- `RapidVisual / CannonVisual / FrostVisual`
  - 对应三种塔的视觉分层
- `SelectionRing`
  - 选中高亮
- `RangeIndicator`
  - 射程圈，默认隐藏

## 3. BuildSpot.prefab

推荐结构：

```text
BuildSpot
  AvailableVisual
  OccupiedVisual
  BuildModeHintVisual
```

推荐挂载：

- `BuildSpot`
- `BuildSpotClickHandler`
- `BuildSpotView`

说明：

- `AvailableVisual`
  - 空闲状态
- `OccupiedVisual`
  - 已建塔状态
- `BuildModeHintVisual`
  - 进入建造模式后显示

## 4. Bullet.prefab

推荐结构：

```text
Bullet
  RapidVisual
  CannonVisual
  FrostVisual
```

推荐挂载：

- `Bullet`
- `BulletView`

说明：

- 三个视觉节点分别对应三种炮塔
- 当前会根据来源炮塔自动切换显示

## 5. HitEffect.prefab

推荐结构：

```text
HitEffect
  CoreVisual
```

推荐挂载：

- `HitEffect`

说明：

- 当前先作为命中反馈占位
- 后续可替换为爆炸、冰冻碎片等更丰富效果

## 命名建议

- 统一使用英文节点名
- 逻辑根节点和脚本类名保持接近
- 可复用的显示节点尽量叫 `Visual / Fill / Root / Ring`

## 当前收益

- 预制体结构更稳定
- Inspector 绑定更清晰
- 后续换美术资源时不需要频繁改脚本
