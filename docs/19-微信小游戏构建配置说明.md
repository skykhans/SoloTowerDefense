# 微信小游戏构建配置说明

## 目标

本文档用于说明当前 Cocos 主线在 Cocos Creator 中构建微信小游戏时，建议关注的关键配置项。

## 一、构建入口

在 Cocos Creator 中：

1. 打开项目
2. 进入“构建发布”
3. 平台选择“微信小游戏”

## 二、建议优先确认的配置

### 1. 场景配置

- 启动场景：
  - 推荐设置为 `HomeScene`
- 构建包含场景：
  - `HomeScene`
  - `MainScene`
  - `ResultScene`

说明：

- 场景名要与 [`SceneNames.ts`](/C:/Test/SoloTowerDefense/assets/scripts/core/SceneNames.ts) 保持一致

### 2. 屏幕方向

建议先明确单一方向，不要一开始兼顾双方向。

推荐：

- 轻策略塔防：优先横屏

说明：

- 一旦方向确定，`HomeScene / MainScene / ResultScene` 的 UI 布局都应按同一方向适配

### 3. 包体与资源策略

前期建议：

- 先不做复杂分包
- 先保证主包可运行
- 贴图和音频资源先用占位资源

后续建议：

- 场景资源按场景拆分
- 大图集和音频做分包或延迟加载

### 4. 调试配置

开发阶段建议：

- 打开调试日志
- 保留错误堆栈
- 在微信开发者工具中启用真机调试

发布前建议：

- 关闭多余调试输出
- 保留必要错误日志

### 5. 本地存储验证

当前无后端阶段，需要重点确认：

- 关卡选择是否能保存
- 最佳波次是否能保存
- 已解锁关卡是否能保存
- 结算后返回首页是否仍能读到最新数据

## 三、构建后目录用途

构建完成后会得到微信小游戏构建产物目录。

建议检查：

- 是否包含三张场景资源
- 是否包含预制体资源
- 是否包含脚本与配置
- 是否能被微信开发者工具正常导入

## 四、当前主线推荐构建顺序

1. 完成 [Cocos场景搭建清单](/C:/Test/SoloTowerDefense/docs/18-Cocos场景搭建清单.md)
2. 在 Cocos Creator 中运行场景
3. 修完场景接线错误
4. 再执行微信小游戏构建
5. 用微信开发者工具打开构建结果
6. 再做真机验证

## 五、当前不建议过早投入的配置

- 复杂资源分包
- 云开发
- 排行榜
- 激励广告
- 多套渠道配置

原因：

- 当前主线还处于单机可玩闭环阶段
- 先把场景、预制体和单局体验跑顺更重要
