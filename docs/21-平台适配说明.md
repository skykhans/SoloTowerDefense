# 平台适配说明

## 目标

本文档说明当前 Cocos 主线中的平台适配层设计，用于统一微信小游戏环境与非微信环境之间的差异。

## 当前目录

位置：

```text
assets/scripts/platform/
```

当前文件：

- `PlatformTypes.ts`
- `PlatformService.ts`
- `WeChatStorageAdapter.ts`
- `WebStorageAdapter.ts`
- `MemoryStorageAdapter.ts`
- `SystemInfoService.ts`
- `VibrationService.ts`
- `ShareService.ts`
- `UILayoutService.ts`

## 当前职责

### 1. 运行环境识别

由：

- `PlatformService.ts`

统一识别当前运行环境是：

- `wechat-minigame`
- `web`
- `unknown`

### 2. 统一存储入口

当前本地存储不再由业务层直接访问：

- `wx.getStorageSync`
- `window.localStorage`

而是统一走：

- `PlatformService.getProfile().storage`

### 3. 兜底回退

当前适配策略：

- 微信小游戏环境
  - 使用 `WeChatStorageAdapter`
- 浏览器环境
  - 使用 `WebStorageAdapter`
- 其他环境
  - 使用 `MemoryStorageAdapter`

这样可以保证：

- 代码在不同环境下都能跑
- 业务层不需要关心具体平台 API

## 当前已经接入的平台能力

- 本地存储
- 系统信息读取
- 震动能力入口
- 分享能力入口
- 布局模式与安全区占位入口

当前已接入业务点：

- `LocalStorageService.ts`
- `HomeUI.ts`
- `ResultUI.ts`
- `BuildSpotClickHandler.ts`
- `TowerPanel.ts`
- `GameManager.ts`
- `PlatformDebugPanel.ts`

当前启动阶段摘要输出：

- `StartupLogService.ts`

## 为什么要单独做平台层

原因：

- 避免业务逻辑里到处直接写 `wx`
- 避免每个系统都自己判断是不是微信环境
- 后续扩展分享、震动、登录、系统信息时有统一入口
- 更容易做 Web 预览和非微信环境调试

## 后续建议扩展

### 第一批

- 系统信息读取
- 震动能力
- 分享占位接口

当前已完成这一批的统一入口封装。

### 布局适配占位

- `UILayoutService.ts`
- `SafeAreaPanel.ts`
- `UILayoutController.ts`
- `AdaptiveLayoutGroup.ts`
- `UILayoutPresets.ts`

说明：

- `UILayoutService.ts` 位于 `platform/`
- `SafeAreaPanel.ts` 与 `UILayoutController.ts` 位于 `ui/`

当前用于：

- 统一输出 `compact / standard` 布局模式
- 为安全区适配提供统一容器
- 读取微信 `safeArea` 和 Web 安全区内边距
- 通过 `Widget` 偏移应用安全区，而不是直接改容器尺寸
- 统一处理紧凑布局缩放与横竖屏节点显隐
- 统一处理容器横竖排切换、间距与内边距调整
- 为常见场景提供默认布局预设，减少手动调参

### 第二批

- 登录能力
- 云开发能力
- 广告能力

### 第三批

- 监控和日志上报
- 多渠道差异封装

## 当前约束

- 业务层尽量不要直接散落使用 `wx`
- 新的平台能力优先放到 `platform/` 目录
- 具体平台 API 通过适配器或服务类向外暴露
- 平台摘要与启动日志可统一通过 `StartupLogService` 输出
