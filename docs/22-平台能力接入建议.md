# 平台能力接入建议

## 目标

本文档说明当前已落地的平台能力入口，推荐在什么场景和业务节点使用。

## 当前可用能力

### 1. 系统信息

入口：

- [`SystemInfoService.ts`](../assets/scripts/platform/SystemInfoService.ts)

推荐用途：

- 启动时读取屏幕尺寸
- 判断 UI 适配策略
- 记录当前运行平台信息

建议接入点：

- `HomeScene` 初始化
- `MainScene` 布局自适应

当前已接入点：

- `HomeUI`
- `PlatformDebugPanel`

推荐辅助能力：

- `StartupLogService`
  - 用于启动时统一输出平台、分辨率、存储模式和存档摘要

### 1.5 布局模式

入口：

- [`UILayoutService.ts`](../assets/scripts/platform/UILayoutService.ts)
- [`UILayoutController.ts`](../assets/scripts/ui/UILayoutController.ts)
- [`AdaptiveLayoutGroup.ts`](../assets/scripts/ui/AdaptiveLayoutGroup.ts)
- [`UILayoutPresets.ts`](../assets/scripts/ui/UILayoutPresets.ts)

推荐用途：

- 判断紧凑布局还是标准布局
- 为安全区容器提供统一入口

建议接入点：

- `HomeUI`
- `PlatformDebugPanel`
- `SafeAreaPanel`
- 首页内容根节点
- 战斗 HUD 容器
- 结算内容容器
- 关卡按钮组
- 战斗页操作按钮组
- 结算页按钮组

当前已接入补充：

- `StartupLogService`
  - 启动时输出安全区边距摘要
- `SafeAreaPanel`
  - 使用 `Widget` 偏移应用安全区，避免重复调整尺寸

### 2. 震动

入口：

- [`VibrationService.ts`](../assets/scripts/platform/VibrationService.ts)

推荐用途：

- 建塔成功时短震动
- 升级炮塔时短震动
- 失败或 Boss 提示时长震动

建议接入点：

- `BuildSpotClickHandler`
- `TowerPanel.onUpgradeTower`
- 失败结算触发处

当前已接入点：

- `BuildSpotClickHandler`
- `TowerPanel`
- `GameManager`

### 3. 分享

入口：

- [`ShareService.ts`](../assets/scripts/platform/ShareService.ts)

推荐用途：

- 结算页分享成绩
- 首页分享游戏
- 后续接“分享得奖励”前的占位能力

建议接入点：

- `ResultUI`
- `HomeUI`

当前已接入点：

- `HomeUI`
- `ResultUI`

## 当前接入原则

- 业务层不直接访问 `wx`
- 统一通过 `platform/` 目录的服务类访问平台能力
- Web 和非微信环境必须允许优雅回退

## 当前推荐优先级

1. 先接 `SystemInfoService`
2. 再接 `VibrationService`
3. 最后给 `ShareService` 做 UI 入口

## 当前注意事项

- 分享能力当前是占位接口
- Web 环境下分享会优雅回退，不代表真实分享成功
- 震动能力要避免频繁触发，优先放关键反馈节点
