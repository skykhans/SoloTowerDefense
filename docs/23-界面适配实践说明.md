# 界面适配实践说明

## 目标

本文档说明当前项目推荐的界面适配组合方式，用于在微信小游戏与 Web 预览环境中统一处理：

- 安全区
- 紧凑布局与标准布局
- 横屏与竖屏切换
- 按钮组和信息栏的横竖排变化

## 三层适配职责

### 第一层：安全区容器

组件：

- [`SafeAreaPanel.ts`](/C:/Test/SoloTowerDefense/assets/scripts/ui/SafeAreaPanel.ts)

职责：

- 读取 `UILayoutService` 输出的安全区内边距
- 通过 `Widget` 偏移保护界面不被刘海、圆角或边缘裁切

推荐挂载位置：

- `HomeScene` 的 `SafeAreaRoot`
- `MainScene` 的 `SafeAreaRoot`
- `ResultScene` 的主要内容根节点

### 第二层：内容区缩放与显隐

组件：

- [`UILayoutController.ts`](/C:/Test/SoloTowerDefense/assets/scripts/ui/UILayoutController.ts)

职责：

- 在 `compact / standard` 模式间切换整体缩放
- 在横竖屏切换时控制特定节点显隐

推荐挂载位置：

- 首页主内容区
- 战斗 HUD 容器
- 结算主面板

推荐用途：

- 小屏设备整体缩小 HUD
- 横屏显示更多信息块
- 竖屏隐藏低优先级调试节点

### 第三层：容器布局切换

组件：

- [`AdaptiveLayoutGroup.ts`](/C:/Test/SoloTowerDefense/assets/scripts/ui/AdaptiveLayoutGroup.ts)

职责：

- 切换 `Layout` 容器的横排或竖排
- 根据布局模式调整 `spacingX / spacingY`
- 根据横竖屏调整 `padding`

推荐挂载位置：

- 首页 `LevelPanel`
- 首页 `ActionPanel`
- 战斗页 `ResourceBar`
- 战斗页 `ActionButtons`
- 战斗页 `BuildToolbar`
- 结算页按钮区

## 推荐组合方式

### HomeScene

1. `SafeAreaRoot` 挂 `SafeAreaPanel`
2. 首页内容根节点挂 `UILayoutController`
3. `LevelPanel` 和 `ActionPanel` 挂 `AdaptiveLayoutGroup`

效果：

- 小屏设备下整体缩放首页内容
- 关卡操作与分享/重置按钮能根据方向切换排列方式

### MainScene

1. `SafeAreaRoot` 挂 `SafeAreaPanel`
2. `TopHUD` 或 HUD 父节点挂 `UILayoutController`
3. `ResourceBar / ActionButtons / BuildToolbar` 挂 `AdaptiveLayoutGroup`

效果：

- 先保证 HUD 在安全区内
- 再根据设备紧凑程度缩放 HUD
- 最后让资源条、操作按钮和建造栏自己切换排列方向

### ResultScene

1. `ResultRoot` 或 `ResultPanel` 挂 `UILayoutController`
2. 结算按钮区挂 `AdaptiveLayoutGroup`

效果：

- 结算内容在小屏设备下不拥挤
- 结果页按钮能在横竖屏间自动切换布局

## 推荐原则

- 安全区保护优先于缩放
- 缩放优先于按钮组细节排列
- 业务脚本尽量不要直接处理布局判断
- 布局逻辑优先收口到通用组件，不要散落在 `HomeUI / BattleUI / ResultUI`

## 当前已支持

- 微信 `safeArea` 读取
- Web 安全区探测
- 启动日志输出安全区摘要
- 调试面板显示平台、分辨率和安全区边距
- 安全区容器在画布尺寸变化时自动刷新
- 布局控制器在画布尺寸变化时自动刷新
- 自适应布局组在画布尺寸变化时自动刷新

## 当前建议的验收方法

- 在 Cocos Creator 中手动切换不同 Game 视图分辨率
- 在微信开发者工具中切换机型预览
- 检查顶部 HUD 是否被裁切
- 检查首页和结算页按钮是否仍然可点击
- 检查按钮组在横竖屏切换时是否出现重叠
