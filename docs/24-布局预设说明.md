# 布局预设说明

## 目标

本文档说明当前 `UILayoutPresets` 提供的默认布局预设，用于减少在 Cocos Creator 中为不同场景手工调整参数的成本。

位置：

- [`UILayoutPresets.ts`](/C:/Test/SoloTowerDefense/cocos-wechat/assets/scripts/ui/UILayoutPresets.ts)

## 适用组件

- [`UILayoutController.ts`](/C:/Test/SoloTowerDefense/cocos-wechat/assets/scripts/ui/UILayoutController.ts)
- [`AdaptiveLayoutGroup.ts`](/C:/Test/SoloTowerDefense/cocos-wechat/assets/scripts/ui/AdaptiveLayoutGroup.ts)

## `UILayoutController` 预设

### `home-content`

推荐挂载：

- `HomeRoot`
- 首页主内容区

特点：

- 紧凑布局下整体缩小到 `0.92`
- 横屏下略微收缩，避免首页过宽

### `battle-hud`

推荐挂载：

- `TopHUD`
- HUD 父容器

特点：

- 紧凑布局下收缩更明显
- 竖屏下略缩，优先保证 HUD 不遮挡战斗区域

### `result-panel`

推荐挂载：

- `ResultRoot`
- `ResultPanel`

特点：

- 小屏设备下适度缩放
- 横屏下保留更接近标准尺寸

## `AdaptiveLayoutGroup` 预设

### `home-level-actions`

推荐挂载：

- `LevelPanel`

特点：

- 竖屏时纵向堆叠
- 横屏时横向排列

### `home-secondary-actions`

推荐挂载：

- `ActionPanel`

特点：

- 比主操作区更紧凑
- 适合分享、重置、设置等次级按钮

### `battle-resource-bar`

推荐挂载：

- `ResourceBar`

特点：

- 横屏时横向展示资源信息
- 竖屏时纵向堆叠，降低顶栏拥挤

### `battle-action-buttons`

推荐挂载：

- `ActionButtons`

特点：

- 横屏时按钮横向排布
- 紧凑布局下自动缩小间距

### `battle-toolbar`

推荐挂载：

- `BuildToolbar`

特点：

- 默认保持横向工具栏
- 小屏设备下压缩按钮间距

### `tower-panel-actions`

推荐挂载：

- 炮塔面板按钮区

特点：

- 默认纵向堆叠升级和出售按钮
- 适合侧边栏或底部浮层

### `result-actions`

推荐挂载：

- 结算按钮区

特点：

- 竖屏时纵向堆叠
- 横屏时切换为横向并排

## 推荐落地方式

### HomeScene

- `HomeRoot` 使用 `home-content`
- `LevelPanel` 使用 `home-level-actions`
- `ActionPanel` 使用 `home-secondary-actions`

### MainScene

- `TopHUD` 使用 `battle-hud`
- `ResourceBar` 使用 `battle-resource-bar`
- `ActionButtons` 使用 `battle-action-buttons`
- `BuildToolbar` 使用 `battle-toolbar`
- 炮塔面板按钮区使用 `tower-panel-actions`

### ResultScene

- `ResultRoot` 或 `ResultPanel` 使用 `result-panel`
- 结算按钮区使用 `result-actions`

## 当前原则

- 预设是默认值，不是强制值
- 如果场景有明显不同的视觉方案，可以切回 `custom`
- 优先先用预设跑通，再做局部微调
