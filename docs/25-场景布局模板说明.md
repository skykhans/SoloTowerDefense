# 场景布局模板说明

## 目标

本文档说明 `SceneLayoutProfiles` 提供的场景级布局模板，用于把：

- `UILayoutController`
- `AdaptiveLayoutGroup`
- `UILayoutPresets`

三者组合成可直接在 `HomeScene / MainScene / ResultScene` 中落地的推荐挂载方案。

位置：

- [`SceneLayoutProfiles.ts`](/C:/Test/SoloTowerDefense/cocos-wechat/assets/scripts/ui/SceneLayoutProfiles.ts)
- [`SceneNodeNames.ts`](/C:/Test/SoloTowerDefense/cocos-wechat/assets/scripts/ui/SceneNodeNames.ts)

## 作用

相比单独看预设表，场景布局模板多了一层“节点级建议”：

- 哪个节点该挂什么组件
- 推荐使用哪个 `presetId`
- 这个节点这样配的目的是什么

适合用于：

- 进 Cocos Creator 搭场景时对照
- 代码评审时核对场景接线是否偏离默认方案
- 后续新增场景时复制现有模板

## HomeScene

推荐模板：

- `HomeRoot`
  - `UILayoutController`
  - `presetId = home-content`
- `LevelPanel`
  - `AdaptiveLayoutGroup`
  - `presetId = home-level-actions`
- `ActionPanel`
  - `AdaptiveLayoutGroup`
  - `presetId = home-secondary-actions`

推荐目的：

- 首页整体缩放统一控制
- 主操作区和次级操作区分开适配

## MainScene

推荐模板：

- `TopHUD`
  - `UILayoutController`
  - `presetId = battle-hud`
- `ResourceBar`
  - `AdaptiveLayoutGroup`
  - `presetId = battle-resource-bar`
- `ActionButtons`
  - `AdaptiveLayoutGroup`
  - `presetId = battle-action-buttons`
- `BuildToolbar`
  - `AdaptiveLayoutGroup`
  - `presetId = battle-toolbar`
- `TowerPanelActions`
  - `AdaptiveLayoutGroup`
  - `presetId = tower-panel-actions`

推荐目的：

- 先保证 HUD 整体缩放合理
- 再分别控制资源栏、操作按钮、建造栏和炮塔面板按钮区

## ResultScene

推荐模板：

- `ResultRoot`
  - `UILayoutController`
  - `presetId = result-panel`
- `ResultActions`
  - `AdaptiveLayoutGroup`
  - `presetId = result-actions`

推荐目的：

- 结算主内容和按钮区分层适配
- 小屏设备下避免按钮与结果文本互相挤压

## 推荐使用方式

1. 先按 `SceneLayoutProfiles` 把推荐节点挂齐
2. 再在 Cocos Creator 中验证横竖屏和不同分辨率
3. 如某个场景视觉风格不同，再从 `custom` 开始微调

## 当前原则

- 模板是默认推荐，不强制绑定
- 优先保持三个场景的适配风格一致
- 新增场景时，优先新增模板，再新增预设
