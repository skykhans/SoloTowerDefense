# 预制体结构自检说明

## 目标

本文档说明当前项目如何对核心预制体做结构自检，尽量在 `MainScene` 加载期就发现：

- 预制体根节点命名不一致
- 关键视觉节点缺失
- 文档结构和预制体真实结构不一致

## 当前覆盖的预制体

- `Enemy.prefab`
- `Tower.prefab`
- `BuildSpot.prefab`
- `Bullet.prefab`
- `HitEffect.prefab`

## 当前校验内容

### 1. 根节点名称

例如：

- `Enemy.prefab` 根节点应为 `Enemy`
- `Tower.prefab` 根节点应为 `Tower`
- `BuildSpot.prefab` 根节点应为 `BuildSpot`

### 2. 关键结构节点

例如：

- `Enemy.prefab` 需要有 `Body / NormalVisual / FastVisual / HeavyVisual / BossVisual / HpBarRoot / HpBarFill`
- `Tower.prefab` 需要有 `Base / VisualRoot / RapidVisual / CannonVisual / FrostVisual / SelectionRing / RangeIndicator`
- `BuildSpot.prefab` 需要有 `AvailableVisual / OccupiedVisual / BuildModeHintVisual`

### 3. 基础脚本组件

`MainSceneInstaller` 仍会继续检查这些基础组件是否挂载：

- `Enemy`
- `Tower`
- `BuildSpot`
- `Bullet`
- `HitEffect`

## 当前收益

- 预制体主体结构是否齐全，可以在进入战斗前尽早暴露
- 预制体命名、结构说明和场景接线形成闭环
- 后续换美术时，不容易把关键结构节点删掉却没发现

## 推荐使用顺序

1. 按 [16-预制体视觉结构规范.md](/C:/Test/SoloTowerDefense/docs/16-预制体视觉结构规范.md) 搭预制体
2. 按 [12-预制体接入说明.md](/C:/Test/SoloTowerDefense/docs/12-预制体接入说明.md) 绑定脚本字段
3. 将预制体拖到 `MainSceneInstaller` 相关 Inspector 字段
4. 运行场景，先看预制体结构自检，再看业务逻辑

## 当前限制

- 只能校验关键节点是否存在
- 还不能自动判断图片、美术样式和碰撞大小是否合理
- 还不能替代真实运行时测试
