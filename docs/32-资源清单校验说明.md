# 资源清单校验说明

## 目标

本文档说明当前项目如何根据统一资源清单，对场景、预制体和音频占位资源做静态校验。

## 相关代码

- [`ResourceManifest.ts`](../assets/scripts/config/ResourceManifest.ts)
- [`ResourceManifestValidator.ts`](../assets/scripts/config/ResourceManifestValidator.ts)
- [`ConfigService.ts`](../assets/scripts/config/ConfigService.ts)

### 场景资源

- `failure.wav`

## 当前工作方式

`ResourceManifestValidator` 会根据 `ResourceManifest` 生成所有期望的相对路径，然后和当前已有资源列表做对比，返回：

- 哪些资源存在
- 哪些资源缺失
- 总缺失数量

当前仓库还提供了可直接运行的本地脚本：

- [`tools/check-resource-manifest.mjs`](../tools/check-resource-manifest.mjs)
- [`tools/check-resource-manifest.ps1`](../tools/check-resource-manifest.ps1)

推荐优先使用跨平台 Node 版本：

```bash
npm run check
```

如果只想单独检查资源文件：

```bash
npm run check:resources
```

PowerShell 版本仍可用于 Windows 环境：

```powershell
./tools/check-resource-manifest.ps1
```

它们都可用于在未安装 Cocos Creator 的阶段，先检查首版占位资源是否已经开始落地。

## 当前收益

- 最小可运行资源清单不再只是文档约定
- 后续可以很容易接入启动检查、调试面板或编辑器工具
- 场景、预制体、音频三类资源都开始走统一入口

## 当前仓库状态

基于当前仓库检查脚本：

- 期望资源总数：`40`
- 当前缺失：`0`

说明：

- 首批最小占位资源已经全部创建完成

## 当前限制

- 目前只校验文件路径是否存在
- 还不会校验图片内容、尺寸、格式是否合理
- 还没有直接接入 Cocos 编辑器面板
